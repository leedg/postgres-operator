📦 Operator Manager:
  kubectl get deployment -n {{ .Release.Namespace }} {{ include "postgres-operator.fullname" . }}-controller-manager

📋 ServiceAccount + RBAC:
  kubectl get sa,clusterrole,clusterrolebinding -n {{ .Release.Namespace }} -l app.kubernetes.io/instance={{ .Release.Name }}

🧬 CRDs (PostgresCluster, BackupJob, ScheduledBackup, Pooler):
  kubectl get crd postgresclusters.postgres.keiailab.io
  kubectl get crd backupjobs.postgres.keiailab.io
  kubectl get crd scheduledbackups.postgres.keiailab.io
  kubectl get crd poolers.postgres.keiailab.io

CRD lifecycle을 별도로 관리하려면 설치 시 Helm 기본 옵션을 사용:
  helm install {{ .Release.Name }} keiailab/postgres-operator --skip-crds

📜 License (ADR 0003):
   - Operator + plugin SDK: Apache-2.0 (clean).
   - 외부 backend AGPL/BUSL/CSL/SSPL 의존 영구 금지. 자체 분산 SQL 은 RFC 0001~0005
     로 단계 도입.

ℹ️  Vanilla PostgreSQL mode (default, ADR 0001 자체 분산 SQL keystone).
   분산 SQL 기능은 RFC 0001 (PostgresCluster v2) + RFC 0002 (ShardRange CRD) +
   RFC 0004 (stateless QueryRouter) + RFC 0005 (distributed transactions) 의
   단계 도입을 추적.
